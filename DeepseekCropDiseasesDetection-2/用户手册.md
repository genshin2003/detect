## 1. 系统概述

本系统用于**农作物叶片/植株等图像的目标检测与病害识别**，典型流程为：用户通过 Web 上传图片、视频或调用摄像头，后端使用 **YOLO（Ultralytics）** 模型推理，可选调用 **DeepSeek** 或 **通义千问（Qwen）** 大语言模型，根据检测类别生成**种植与防治方面的文字建议**。检测结果与媒体文件可持久化到 MySQL，并在「识别记录」页面查询与管理。

**单机部署**指：浏览器、Vue 开发服务、Spring Boot、Flask、MySQL 均运行在同一台计算机上，使用 `localhost` / `127.0.0.1` 互相访问。本文默认 **Windows** 环境说明常见路径；Linux/macOS 下命令需相应替换。

## 2. 架构与组件说明

| 组件 | 目录 | 技术栈 | 默认端口 | 主要职责 |
|------|------|--------|----------|------------|
| 前端 | `CropDiseasesDetection_vue` | Vue 3、Vite、Element Plus、Pinia 等 | **8888** | 界面、上传、列表、代理 API |
| 后端 | `CropDiseasesDetection_springboot` | Spring Boot 2.3.7、MyBatis-Plus、MySQL | **9999** | 用户、文件存储、记录 CRUD、转发图片预测到 Flask |
| 推理服务 | `CropDiseasesDetection_flask` | Flask、Flask-SocketIO、Ultralytics YOLO、OpenCV | **5000** | 模型列表、单图/批量/视频/摄像头推理、可选 LLM 建议 |

**数据流简述：**

- 单张**图片检测**：Vue → Spring `/flask/predict` → Flask `POST /predictImg` → 结果写库 `/imgRecords`（由 Spring 在成功时插入）。
- **批量图片**：Vue 将 ZIP 上传到 Spring `/files/uploadFolder`，再请求 Flask `POST /predictImgBatch`（经 Vite 代理 `/flask/...`）。
- **视频 / 摄像头**：浏览器直连或经配置访问 Flask 流式地址；处理结束后 Flask 通过 HTTP 将 JSON 提交到 Spring 的 `/videoRecords` 或 `/cameraRecords`。
- **WebSocket**：图像检测等场景使用 Socket.IO 连接 **Flask 5000**（如进度、AI 生成提示），需保证防火墙未拦截。

## 3. 单机部署环境要求

### 3.1 硬件建议

- **CPU**：四核及以上；推理以 CPU 亦可运行，速度随核数与模型大小变化。
- **内存**：建议 **8GB** 及以上；视频批处理、大模型同时加载时更占内存。
- **磁盘**：预留 **10GB+**（含依赖、`node_modules`、上传文件、`weights` 模型）。
- **GPU（可选）**：若已安装 CUDA 对应版本的 PyTorch，Ultralytics 可自动使用 GPU 加速；未配置 GPU 时使用 CPU 即可。

### 3.2 软件版本建议

| 软件 | 建议版本 | 说明 |
|------|----------|------|
| 操作系统 | Windows 10/11 64 位 | 单机文档以 Windows 为主 |
| JDK | 17 | 与 `pom.xml` 中 `java.version` 一致 |
| Maven | 3.6+ | 构建 Spring Boot |
| Node.js | **≥ 16** | 与 `package.json` 中 `engines` 一致 |
| npm | **≥ 7** | 随 Node 安装 |
| MySQL | ** 8.0** 等 | 与驱动兼容即可；脚本含 utf8mb4 |
| Python | **3.12** 推荐 | 需与 PyTorch、Ultralytics 版本匹配 |
| FFmpeg | 已加入 **PATH** | 视频转 MP4 必需；仓库根目录有 `ffmpeg-7.1-full_build` 可参考安装 |

## 5. MySQL 数据库安装与导入

### 5.1 创建数据库

Spring 默认连接（`CropDiseasesDetection_springboot/src/main/resources/application.properties`）：

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/yolo_detect?serverTimezone=Asia/Shanghai
spring.datasource.username=root
spring.datasource.password=123456
```

在 MySQL 中执行（密码请按实际修改）：

```sql
CREATE DATABASE IF NOT EXISTS yolo_detect
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;
```

### 5.2 导入 SQL 脚本

项目根目录文件：**`CropDiseasesDetection.sql`**

使用命令行或 Navicat / MySQL Workbench 将脚本导入到 **`yolo_detect`** 库。

**注意：** SQL 文件头部注释可能写「源库为 test」，与 Spring 配置的库名 **`yolo_detect`** 不一致时，务必导入到 **`yolo_detect`**，或统一修改 `application.properties` 中的库名。

### 5.3 默认测试账号（以脚本中数据为准）

导入脚本后，通常存在管理员账号：

| 用户名 | 密码 | 角色 |
|--------|------|------|
| `admin` | `admin` | `admin` |

其他测试用户（如 `1111/111`、`222/222`）以 SQL 中 `INSERT INTO user` 为准。生产环境请**立即修改密码**并删除无用测试账号。

### 5.4 主要数据表

- **`user`**：用户账号、角色、头像等  
- **`imgrecords`**：图片检测记录（输入图、结果图、标签、置信度、AI 建议等）  
- **`videorecords`**：视频检测记录  
- **`camerarecords`**：摄像头检测记录

## 6. Spring Boot 后端部署

### 6.1 修改配置

编辑：`CropDiseasesDetection_springboot/src/main/resources/application.properties`

单机本机建议保持：

- `server.port=9999`
- `spring.datasource.url` 中主机 `localhost`、库名 `yolo_detect`
- `spring.datasource.username` / `password` 与本地 MySQL 一致

大文件上传上限已配置为 **500MB**，可按需调整：

```properties
spring.servlet.multipart.max-file-size=500MB
spring.servlet.multipart.max-request-size=500MB
```

### 6.2 编译与运行

在目录 `CropDiseasesDetection_springboot` 下：

```powershell
mvn clean package -DskipTests
java -jar target\Kcsj-0.0.1-SNAPSHOT.jar
```

或在 IDE（IntelliJ IDEA）中运行主类：**`com.example.Kcsj.Kcsj`**（以 `pom.xml` 中 `spring-boot-maven-plugin` 的 `mainClass` 为准）。

### 6.3 验证

浏览器访问：`http://localhost:9999/`  
若需验证接口，可用 Postman 调用 `POST http://localhost:9999/user/login`，Body JSON 示例：`{"username":"admin","password":"admin"}`。

上传的文件默认保存在项目运行目录下的 **`files/`** 文件夹（由 `FileController` 使用 `System.getProperty("user.dir")` 拼接路径）。

## 7. Flask 推理服务部署

### 7.1 进入目录与虚拟环境

```powershell
cd CropDiseasesDetection_flask
python -m venv venv
.\venv\Scripts\activate
```

### 7.2 安装 Python 依赖

仓库若未附带 `requirements.txt`，请至少安装（版本可按环境微调）：

```powershell
pip install flask flask-socketio ultralytics opencv-python-headless requests
```

若使用 GPU，请按 [PyTorch 官网](https://pytorch.org/) 选择对应 CUDA 版本安装 `torch`。

### 7.3 模型权重

将 `.pt`、`.onnx` 等模型文件放入：

**`CropDiseasesDetection_flask/weights/`**

接口 `GET /file_names` 会扫描该目录，前端下拉框显示可选模型。

### 7.4 配置 API Key（可选）

编辑 **`CropDiseasesDetection_flask/main.py`**，将占位符改为真实密钥：

```python
self.DeepSeek = '你的DeepSeek密钥'
self.Qwen = '你的Qwen密钥'
```

若不配置，选择「不使用 AI」仍可正常检测；选择 DeepSeek/Qwen 时需有效 Key 与网络。

### 7.5 与 Spring 的地址（单机必看）

`main.py` 中 `save_data` 调用写死了：

- `http://localhost:9999/videoRecords`
- `http://localhost:9999/cameraRecords`

单机且 Spring 在 **9999** 端口时无需修改。若更改 Spring 端口或主机名，必须同步修改此处。

### 7.6 启动服务

```powershell
python main.py
```

默认监听 **`0.0.0.0:5000`**，本机浏览器可通过 `http://127.0.0.1:5000` 访问（具体路由以前端为准）。

### 7.7 FFmpeg

视频、摄像头录制后经 FFmpeg 转为 MP4（`utils/Fun.py`）。若转换失败，请检查：

1. 命令行能执行 `ffmpeg`  
2. 必要时将仓库内 `ffmpeg-7.1-full_build\bin` 加入系统 **PATH**

## 8. Vue 前端部署

### 8.1 安装依赖

```powershell
cd CropDiseasesDetection_vue
npm install
```

### 8.2 开发模式（单机推荐调试用）

```powershell
npm run dev
```

根据 **`CropDiseasesDetection_vue/.env`**：

- 开发服务器端口：**`VITE_PORT = 8888`**
- 浏览器访问：**`http://localhost:8888`**

`vite.config.ts` 中已配置代理：

- **`/api`** → `http://localhost:9999/`（去掉 `/api` 前缀后转发）
- **`/flask`** → `http://localhost:5000/`（去掉 `/flask` 前缀后转发）

因此前端代码里使用 **`/api/user/login`** 实际会转到 Spring 的 **`/user/login`**。

### 8.3 Axios 基地址（`VITE_API_DOMAIN`）

`src/utils/request.ts` 使用 `import.meta.env.VITE_API_DOMAIN` 作为 `baseURL`。

- 若未定义：相对路径请求会发到当前前端站点（8888），依赖上述代理，**单机开发通常可用**。
- 生产构建建议在 **`.env.production`** 中显式设置，例如与部署站点同源的相对根路径或完整 API 地址，避免环境不一致。

### 8.4 写死的本机地址（单机可不改，换机必改）

以下文件中含 **`localhost`** 或 **`127.0.0.1`**，仅在本机访问时一般正常；若改为局域网 IP 部署，需批量替换并重新构建：

| 用途 | 示例位置 |
|------|----------|
| 文件上传 | `imgPredict/index.vue`、`videoPredict/index.vue`、`personal/index.vue`、`userManage/dialog.vue` 等 `action="http://localhost:9999/files/upload"` |
| 文件夹 ZIP 上传 | `imgPredictBatch/index.vue` 中 `http://localhost:9999/files/uploadFolder` |
| 视频/摄像头流 | `videoPredict/index.vue`、`cameraPredict/index.vue` 中 `http://127.0.0.1:5000/predictVideo`、`predictCamera` |
| Socket.IO | `src/utils/socket.ts` 中 `http://localhost:5000` |
| 视频记录外链 | `videoRecord/index.vue` 中 `http://localhost:8888/#/videoShow?...` |

### 8.5 生产构建（可选）

```powershell
npm run build
```

产物在 **`dist/`**。需将 `dist` 部署到 Nginx 等静态服务器，并配置：

- 反向代理 **`/api`** 到 Spring（如 `http://127.0.0.1:9999`）
- 反向代理 **`/flask`** 到 Flask（如 `http://127.0.0.1:5000`）

## 10. 功能说明（按菜单）

以下与 `CropDiseasesDetection_vue/src/router/route.ts` 中菜单一致。

### 10.1 首页

- 展示与统计相关的数据拉取（如调用 `/api/imgRecords/all` 等），便于总览。

### 10.2 图像检测

- **选择模型**：对应 `weights` 目录下文件。  
- **选择 AI 助手**：可不使用、DeepSeek、Qwen（需在 Flask 配置 Key）。  
- **最小置信度阈值**：滑块数值会参与推理（具体换算以前端与后端约定为准，常见为除以 100 得到 0～1）。  
- **上传图片**：上传到 Spring `POST /files/upload`，得到 URL 后再走预测。  
- **开始预测**：调用 Spring `POST /flask/predict`，内部转发 Flask。  
- **PDF 导出**：将当前页面区域导出为 PDF。  
- **Socket 提示**：检测完成或生成 AI 建议时可能通过 Socket.IO 推送消息。

### 10.3 批量检测

- 将多张图片打包为 **ZIP** 上传至 `POST /files/uploadFolder`，后端解压到 `files/{uuid}/`。  
- 调用 Flask **`POST /predictImgBatch`**（请求体含 `imgFolderUrl`、`username`、`weight`、`conf` 等）。  
- 结果以列表展示，可切换查看各张结果。

### 10.4 视频检测

- 视频先上传到 Spring，再拼接查询参数访问 Flask **`GET /predictVideo`** 流式预览。  
- 处理结束后 Flask 上传结果并 `POST` 到 **`/videoRecords`**。  
- 在 **「视频识别记录」** 中查看；部分功能会打开 `#/videoShow` 播放（地址写死为 localhost:8888 时需本机前端运行）。

### 10.5 摄像检测

- 使用本机摄像头，连接 Flask **`GET /predictCamera`**。  
- **`GET /stopCamera`** 用于停止循环采集。  
- 结束后记录写入 **`/cameraRecords`**。

### 10.6 图片 / 视频 / 摄像识别记录

- 分页列表、条件筛选（用户名、时间、标签、置信度等字段依页面为准）。  
- 支持删除等操作（具体以界面按钮为准，对应 `DELETE /api/imgRecords/{id}` 等）。

### 10.7 用户管理（仅管理员）

- 路由 `roles: ['admin']`。  
- 用户列表、新增、编辑、删除；头像上传走 `/files/upload`。

### 10.8 个人中心

- 查看与修改当前用户信息；头像上传等。

### 10.9 角色与菜单可见性

路由 `meta.roles` 控制前端是否注册该路由（与 `frontEnd.ts` 等逻辑配合）。常见角色：

- **`admin`**：含用户管理与全部业务菜单。  
- **`common`**、**`others`**：普通用户使用检测与记录功能。  
- **`specialist`**：在路由配置中用于首页等，若需与业务菜单一致，可在 `route.ts` 中扩展 `roles` 数组。

## 13. 常见问题与排错

### 13.1 登录成功但菜单不全

检查该用户在 **`user`** 表中的 **`role`** 字段是否为 `admin` / `common` 等与路由 `roles` 一致。

### 13.2 模型下拉为空

- Flask 是否启动；`weights` 目录是否有文件。  
- 浏览器 F12 查看 `/api/flask/file_names` 是否 404 或 500。  
- Spring 是否能访问 `http://127.0.0.1:5000/file_names`。

### 13.3 图片预测报「Error」或连接失败

- 确认 Spring 与 Flask 均已启动。  
- `PredictionController` 中调用的是 `http://localhost:5000/predictImg`，若 Flask 仅绑定或防火墙异常会失败。

### 13.4 视频处理卡住或无法生成 MP4

- 本机执行 `ffmpeg -version` 是否正常。  
- 查看 Flask 控制台是否有 FFmpeg 报错。  
- 磁盘空间是否充足。

### 13.5 视频/摄像头记录未入库

- Spring 是否在 **9999**；Flask 中 `save_data` URL 是否与之一致。  
- 网络策略是否拦截 Flask 对本机 9999 的 POST。

### 13.6 上传文件 413 或过大失败

在 `application.properties` 中增大 `max-file-size` / `max-request-size`，并检查反向代理（若有）的 body 限制。

### 13.7 数据库连接失败

核对 URL、用户名、密码、MySQL 是否监听 `3306`，以及库名是否为 **`yolo_detect`**。
