# 前端文件使用说明（Vue3 + Vite 项目）

对应目录：`CropDiseasesDetection_vue/`

你不熟悉前端时，建议按这个顺序理解：

- **先看入口与代理**：`vite.config.ts`、`src/main.ts`、`src/utils/request.ts`
- **再看页面路由**：`src/router/route.ts`
- **最后看具体业务页面**：`src/views/*`

---

## 1. 如何启动与“请求到底打到哪里”

### 1.1 启动（开发模式）

在 `CropDiseasesDetection_vue` 目录：

- 安装依赖：`npm install`
- 启动：`npm run dev`

默认会打开：`http://localhost:8888`

### 1.2 代理规则（最重要的前端配置）

文件：`CropDiseasesDetection_vue/vite.config.ts`

- **`/api/*` → Spring Boot `http://localhost:9999/`**（并去掉 `/api` 前缀）
- **`/flask/*` → Flask `http://localhost:5000/`**（并去掉 `/flask` 前缀）

因此：

- 前端写 `request.get('/api/imgRecords')` ⇒ 实际访问 `http://localhost:9999/imgRecords`
- 前端写 `request.post('/flask/predictImgBatch')` ⇒ 实际访问 `http://localhost:5000/predictImgBatch`

> 注意：项目里也有直接写死的绝对地址（例如上传 `http://localhost:9999/files/upload`、视频流 `http://127.0.0.1:5000/predictVideo?...`），这些不经过代理。

## 2. 目录结构总览（你需要重点关注的）

`src/` 的关键目录：

- `src/main.ts`：Vue 应用入口（注册 pinia、router、ElementPlus、i18n）

- `src/theme/`：主题样式（scss）
- `src/assets/`：静态资源（svg）
- `src/components/`：通用组件（表格、svg 图标等）
- `src/i18n/`：国际化资源（本项目默认中文）
- `src/types/`：TS 类型声明（让 IDE/编译通过）

## 3. 每个文件做什么（按目录逐个说明）

下面是基于当前仓库 `src` 下文件列表（约 129 个）逐个说明。  
说明原则：**业务相关文件会写“它调用了哪些接口/服务”，模板/框架文件会写“它提供什么能力/通常不用改”。**

### 3.1 入口与全局

- **`src/main.ts`**：应用入口。注册 `pinia`、`router`、`ElementPlus`、`i18n`、`VueGridLayout`，并挂载到 `#app`。
- **`src/App.vue`**：根组件。提供 ElementPlus 全局配置、锁屏组件、设置抽屉、全屏关闭按钮；监听路由变化更新标题等。

### 3.2 网络与工具（`src/utils/`）

- **`src/utils/request.ts`**：Axios 实例与拦截器。  
  - `baseURL` 使用 `import.meta.env.VITE_API_DOMAIN`（实际开发中很多请求走相对路径 + Vite proxy）。  
  - 请求拦截器：从 `Session.get('token')` 设置 `Authorization`。  
  - 响应拦截器：如果 `res.code===401/4001` 清理缓存并跳转登录。

- **`src/utils/socket.ts`**：Socket.IO 客户端封装。默认连接 `http://localhost:5000`，支持 `on/off/emit/disconnect`。  
  - 页面用于接收 Flask 推送：`message`、`progress`。

- **`src/utils/formatTime.ts`**：时间格式化工具。项目里生成 `startTime` 使用 `formatDate()`。
- **`src/utils/pdf.ts`**：将页面指定 DOM 转成 PDF（`html2canvas + jsPDF`）。图片检测页用它导出检测报告。
- **`src/utils/api.ts`**：对 `request()` 的薄封装（通用请求方法）。
- **`src/utils/other.ts`**：杂项工具集合（注册 ElementPlus 图标、设置标题、深拷贝、判断移动端等）。
- **`src/utils/storage.ts`**：浏览器缓存封装（`Local`、`Session`），其中 token 用 cookies 存。
- **`src/utils/mitt.ts`**：事件总线（mitt）封装（用于跨组件通信）。
- **`src/utils/toolsValidate.ts`**：校验工具（如 URL 校验，供菜单外链等功能使用）。
- **`src/utils/setIconfont.ts`**：动态注入 iconfont 等外部资源（在 `App.vue` 里被调用）。
- **`src/utils/getStyleSheets.ts`**：获取样式表/处理主题相关。
- **`src/utils/print.ts`**：打印相关工具（如果存在则为 print-js 封装）。
- **`src/utils/clipboard.ts`**：剪贴板封装（如果存在则为 vue-clipboard3 使用）。

> 说明：`utils` 下有不少来自 `vue-next-admin` 模板的通用能力，业务强相关的主要是 `request.ts`、`socket.ts`、`formatTime.ts`、`pdf.ts`。

### 3.3 路由（`src/router/`）

- **`src/router/route.ts`**：路由表定义。  
  - `dynamicRoutes`：layout 内的页面路由（首页、检测、记录、用户管理、个人中心等）。  
  - `staticRoutes`：全屏页面（登录、注册、视频详情页）。

- **`src/router/index.ts`**：路由实例 + 路由守卫。  
  - 没有 token 会强制跳转 `/login`。  
  - `/videoShow` 在已登录时允许直接进入。  
  - 首次加载会初始化动态路由（前端控制/后端控制两套模式，本项目默认 `isRequestRoutes=false`）。

- **`src/router/frontEnd.ts`**：前端控制路由初始化（将 `dynamicRoutes` 装载到 router）。
- **`src/router/backEnd.ts`**：后端控制路由初始化（模板功能，本项目一般不需要改）。

### 3.4 状态管理（`src/stores/`）

- **`src/stores/index.ts`**：创建 pinia 实例并导出。
- **`src/stores/themeConfig.ts`**：全局主题/布局配置（颜色、暗黑模式、是否折叠菜单、是否请求后端路由等）。你要改整体 UI 风格就看它。
- **`src/stores/userInfo.ts`**：用户信息状态（用户名、角色等）。  
  - 注意：这里的 `getApiUserInfo()` 是模板的“模拟数据”，本项目真实登录逻辑在 `views/login/index.vue`，它会把登录结果写入 `Session`。
- **`src/stores/routesList.ts`**：动态路由列表状态（配合路由初始化）。
- **`src/stores/keepAliveNames.ts`**：keep-alive 缓存页面 name 列表。
- **`src/stores/tagsViewRoutes.ts`**：顶部 tagsView（多标签页）状态。
- **`src/stores/requestOldRoutes.ts`**：后端路由请求时的旧路由缓存（模板功能）。

### 3.5 业务页面（`src/views/`）

#### 3.5.1 登录/注册

- **`src/views/login/index.vue`**：登录页。  
  - 调用：`POST /api/user/login`（最终到 Spring `/user/login`）  
  - 成功后：写入 token、用户信息并跳转首页（细节以文件内逻辑为准）。

- **`src/views/login/register.vue`**：注册页。  
  - 调用：`POST /api/user/register`（最终到 Spring `/user/register`）

#### 3.5.2 首页统计

- **`src/views/home/index.vue`**：首页。  
  - 调用：`GET /api/imgRecords/all` 获取所有图片识别记录，做折线图/饼图/柱状图/雷达图与“实时预测信息”表格展示（ECharts）。

#### 3.5.3 单图检测（最核心）

- **`src/views/imgPredict/index.vue`**：单张图片检测页面。  
  - 上传图片：`action="http://localhost:9999/files/upload"`  
  - 获取模型列表：`GET /api/flask/file_names`（Spring 桥接 Flask）  
  - 发起预测：`POST /api/flask/predict`（到 Spring `/flask/predict`，由 Spring 再调 Flask `/predictImg`）  
  - WebSocket：连接 Flask `5000`，监听 `message`（用于提示“检测完成/生成AI建议中”等）
  - 结果展示：解析 `label/confidence`（它们是 JSON 字符串），并支持导出 PDF（`htmlToPDF()`）。

#### 3.5.4 批量检测

- **`src/views/imgPredictBatch/index.vue`**：批量图片检测页面。  
  - 选择本地文件夹后，用 `JSZip` 打包成 zip  
  - 上传 zip：`POST http://localhost:9999/files/uploadFolder`  
  - 批量预测：`POST /flask/predictImgBatch`（**直连 Flask**，经 Vite 代理）  
  - WebSocket：监听 Flask `message`
  - 注意：当前代码路径下，批量预测结果主要用于前端展示，**并未写入 Spring 的 imgRecords**（这点是调用链上的差异）。

#### 3.5.5 视频检测

- **`src/views/videoPredict/index.vue`**：视频检测页面。  
  - 上传视频：`POST http://localhost:9999/files/upload`
  - 获取模型列表：`GET /api/flask/file_names`
  - 开始处理：直接把 `<img :src>` 指向 `http://127.0.0.1:5000/predictVideo?...`（**直连 Flask 流式接口**）
  - WebSocket：监听 `message/progress` 显示保存进度
  - 视频记录入库：由 Flask 处理完成后回调 Spring `/videoRecords` 完成（前端不直接调用入库接口）。

#### 3.5.6 摄像头检测

- **`src/views/cameraPredict/index.vue`**：摄像头检测页面。  
  - 获取模型列表：`GET /api/flask/file_names`
  - 开始录制：流地址 `http://127.0.0.1:5000/predictCamera?...`
  - 停止录制：`GET /flask/stopCamera`（Vite 代理到 Flask `/stopCamera`）
  - WebSocket：监听 `message/progress`
  - 记录入库：由 Flask 回调 Spring `/cameraRecords`。

#### 3.5.7 识别记录

- **`src/views/imgRecord/index.vue`**：图片识别记录列表页。  
  - 调用分页：`GET /api/imgRecords`（支持搜索 username/startTime/label/conf）  
  - 展开行：把一条记录的 `label/confidence` 拆成明细表  
  - 删除：`DELETE /api/imgRecords/{id}`

- **`src/views/videoRecord/index.vue`**：视频识别记录列表页。  
  - 调用分页：`GET /api/videoRecords`
  - 删除：`DELETE /api/videoRecords/{id}`
  - 查看详情：打开新窗口 `/#/videoShow?id=...`

- **`src/views/videoRecord/show.vue`**：视频记录详情页（左右对比原视频与处理后视频，可拖动分割条）。  
  - 获取详情：`GET /api/videoRecords/{id}`

- **`src/views/cameraRecord/index.vue`**：摄像头识别记录列表页。  
  - 调用分页：`GET /api/cameraRecords`
  - 删除：`DELETE /api/cameraRecords/{id}`

#### 3.5.8 用户与个人中心

- **`src/views/userManage/index.vue`**：用户管理页（管理员可见）。  
  - 分页：`GET /api/user`
  - 删除：`DELETE /api/user/{id}`
  - 添加/编辑：弹窗组件 `dialog.vue`（见下）

- **`src/views/userManage/dialog.vue`**：用户新增/编辑弹窗。  
  - 头像上传：`POST http://localhost:9999/files/upload`
  - 保存/更新：对应调用 `/api/user` 或 `/api/user/update`（以文件内为准）

- **`src/views/personal/index.vue`**：个人中心。  
  - 读取当前用户：`GET /api/user/{username}`
  - 头像上传：`POST http://localhost:9999/files/upload`
  - 更新：`POST /api/user/update`

#### 3.5.9 错误页/其他模板页

## 5. 常见问题（结合本项目）

- **为什么有的请求是 `/api/...` 有的是 `/flask/...`？**  
  因为 Vite 代理把它们分别转发到 Spring(9999) 和 Flask(5000)，这是项目当前采用的混合调用方式。

- **为什么单图预测返回的 `res.data` 还要 `JSON.parse`？**  
  Spring 的 `/flask/predict` 返回的是 `Result.success(responseString)`，其中 `data` 是 Flask 返回的 **JSON 字符串**，所以前端需要二次解析。

- **为什么批量预测不入库？**  
  批量页调用的是 Flask `/predictImgBatch` 并直接在前端展示；当前代码没把批量结果回写到 Spring 的 `/imgRecords`。
