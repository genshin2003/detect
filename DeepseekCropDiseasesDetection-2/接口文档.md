# 农作物病害智能检测系统——接口文档（基于当前代码）

本文档以你当前仓库内代码为准，覆盖以下三部分：

- **Spring Boot 后端（端口 9999）**：用户/文件/记录 CRUD、图片预测转发与入库
- **Flask 推理服务（端口 5000）**：模型列表、单图/批量/视频/摄像头推理、WebSocket 消息
- **前端开发代理（端口 8888）**：Vite 代理规则（决定前端请求最终打到哪里）

---

## 1. 端口与服务

- **前端（Vue/Vite）**：`http://localhost:8888`
- **Spring Boot**：`http://localhost:9999`
- **Flask + Socket.IO**：`http://localhost:5000`
- **MySQL**：`localhost:3306`（库名：`yolo_detect`，见 `CropDiseasesDetection_springboot/src/main/resources/application.properties`）

---

## 2. 前端代理规则（非常关键）

在 `CropDiseasesDetection_vue/vite.config.ts` 中：

- **`/api/*` → 代理到 Spring Boot 9999**，并把前缀 `/api` 去掉  
  - 例：前端请求 `/api/user/login` 实际到 `http://localhost:9999/user/login`
- **`/flask/*` → 代理到 Flask 5000**，并把前缀 `/flask` 去掉  
  - 例：前端请求 `/flask/predictImgBatch` 实际到 `http://localhost:5000/predictImgBatch`

> 这也是为什么你会看到前端代码里既有 `/api/...` 又有 `/flask/...`：它们是两条不同的后端通道。

---

## 3. 统一返回结构（Spring Boot）

Spring Boot 使用 `Result<T>`（`com.example.Kcsj.common.Result`）作为统一返回：

```json
// 成功
{ "code": "0", "msg": "成功", "data": ... }

// 失败
{ "code": "-1", "msg": "错误原因", "data": null }
```

前端一般用 `res.code == 0` 判断成功。

## 4. Spring Boot 接口（9999）

### 4.1 用户模块 `UserController`

基础路径：`/user`

#### 4.1.1 登录
- **POST** `/user/login`
- **Body(JSON)**：

```json
{ "username": "admin", "password": "123456" }
```

- **返回**：`Result<User>`（用户实体，包含 `role/avatar/...` 等字段）
- **前端调用位置**：`CropDiseasesDetection_vue/src/views/login/index.vue`

#### 4.1.2 注册
- **POST** `/user/register`
- **Body(JSON)**：

```json
{ "username": "u1", "password": "p1" }
```

> 后端会写入默认资料（姓名/性别/头像/role=common 等）。

#### 4.1.3 用户分页查询
- **GET** `/user`
- **Query**：
  - `pageNum`（默认 1）
  - `pageSize`（默认 10）
  - `search`（按 username 模糊）

#### 4.1.4 按用户名查询
- **GET** `/user/{username}`

#### 4.1.5 更新
- **POST** `/user/update`
- **Body(JSON)**：User 全量（按 `id` 更新）

#### 4.1.6 删除
- **DELETE** `/user/{id}`

---

### 4.2 文件模块 `FileController`

基础路径：`/files`

#### 4.2.1 单文件上传（图片/视频/头像等）
- **POST** `/files/upload`
- **Content-Type**：`multipart/form-data`
- **表单字段**：`file`
- **返回**：`Result<string>`，其中 `data` 是可访问 URL：

```json
{ "code":"0","msg":"成功","data":"http://localhost:9999/files/<uuid>_<原文件名>" }
```

> Flask 也会用这个接口上传推理输出文件（见 `CropDiseasesDetection_flask/utils/Fun.py` 的 `upload()`）。

#### 4.2.2 文件夹上传（ZIP）
- **POST** `/files/uploadFolder`
- **Content-Type**：`multipart/form-data`
- **表单字段**：`folder`（必须是 `.zip`）
- **返回**：`Result<string>`，`data` 为文件夹 URL：

```json
{ "code":"0","msg":"成功","data":"http://localhost:9999/files/<uuid>" }
```

> 前端批量检测会把选中的文件夹打包成 zip 上传到这里（见 `imgPredictBatch/index.vue`）。

#### 4.2.3 下载：文件夹（打包 zip）或单文件（兼容）
- **GET** `/files/{flag}`
  - 若 `{flag}` 对应 `files/{flag}/` 文件夹，则打包下载 `flag.zip`
  - 若 `{flag}` 对应单文件，则下载该文件
  - 若找不到，会尝试匹配 `{flag}_*` 前缀文件

#### 4.2.4 下载：下载文件夹内指定文件
- **GET** `/files/{flag}/{filename}`

### 4.3 图片识别记录 `ImgRecordsController`

基础路径：`/imgRecords`

#### 4.3.1 分页查询（支持筛选）
- **GET** `/imgRecords`
- **Query**：
  - `pageNum`（默认 1）
  - `pageSize`（默认 10）
  - `search`：按 `username` 模糊
  - `search1`：按 `startTime` 模糊
  - `search2`：按 `lable` 模糊
  - `search3`：按 `conf` 模糊

#### 4.3.2 全量
- **GET** `/imgRecords/all`

#### 4.3.3 详情
- **GET** `/imgRecords/{id}`

#### 4.3.4 新增
- **POST** `/imgRecords`
- **Body(JSON)**：ImgRecords

#### 4.3.5 更新
- **POST** `/imgRecords/update`

#### 4.3.6 删除
- **DELETE** `/imgRecords/{id}`

### 4.6 图片预测桥接 `PredictionController`

基础路径：`/flask`

#### 4.6.1 获取模型列表（桥接 Flask）
- **GET** `/flask/file_names`
- **后端行为**：Spring 调用 `GET http://127.0.0.1:5000/file_names`，并 `Result.success(responseString)`
- **返回**：`Result<string>`，`data` 是一个 JSON 字符串（前端需要 `JSON.parse(res.data)`）

#### 4.6.2 单图预测（桥接 Flask + 写库）
- **POST** `/flask/predict`
- **Body(JSON)**（前端 `imgPredict/index.vue` 发送）：

```json
{
  "startTime": "2026-04-13 12:00:00",
  "weight": "xxx.pt",
  "username": "u1",
  "inputImg": "http://localhost:9999/files/<uuid>_a.jpg",
  "conf": 0.5,
  "ai": "DeepSeek",
  "suggestion": ""
}
```

- **后端行为**：
  - Spring 调用 `POST http://localhost:5000/predictImg`
  - Flask 返回一个 JSON 字符串（含 `status/label/confidence/outImg/allTime/suggestion` 等）
  - 若 `status==200`，Spring 会把结果写入 MySQL（`img_records` 对应实体 `ImgRecords`）
  - 最终 Spring 返回：`Result.success(responseString)`

- **返回**：`Result<string>`，`data` 是 JSON 字符串（前端需要 `JSON.parse(res.data)`，并进一步解析 `label/confidence`）

## 5. Flask 接口（5000）

### 5.1 模型列表
- **GET** `/file_names`
- **返回**（字符串 JSON）：

```json
{ "weight_items": [ { "value": "xxx.pt", "label": "xxx.pt" } ] }
```

### 5.2 单图预测
- **POST** `/predictImg`
- **Body(JSON)**：

```json
{
  "username": "u1",
  "weight": "xxx.pt",
  "conf": 0.5,
  "startTime": "2026-04-13 12:00:00",
  "inputImg": "http://localhost:9999/files/<uuid>_a.jpg",
  "ai": "DeepSeek"
}
```

- **返回**（字符串 JSON，`ensure_ascii=False`）：
  - 成功：`status=200`，包含：
    - `outImg`：推理结果图片 URL（先在本地生成 `./runs/result.jpg`，再上传到 Spring `/files/upload`）
    - `label`：JSON 字符串数组，例如 `["玉米-叶斑病"]`
    - `confidence`：JSON 字符串数组，例如 `["92.31%"]`（具体格式取决于 `predictImg` 模块）
    - `allTime`：耗时
    - `suggestion`：AI 建议（Markdown 文本）
  - 失败：`status=400`，含 `message`

> 前端会把 `suggestion` 用 `marked()` 渲染成 HTML（见 `imgPredict/index.vue`）。

### 5.3 批量图片预测
- **POST** `/predictImgBatch`
- **Body(JSON)**：

```json
{
  "imgFolderUrl": "http://localhost:9999/files/<uuid>",
  "username": "u1",
  "weight": "xxx.pt",
  "conf": 0.5
}
```

- **行为**：Flask 会下载 zip 并解压到 `./runs/imgBatch`，逐张预测，组装批量结果返回
- **返回（对象）**：

```json
{ "code": 0, "message": "预测成功", "data": [ ... ] }
```

> 注意：这个接口前端走的是 `/flask/predictImgBatch`（Vite 代理到 Flask），**不经过 Spring**。

### 5.4 视频预测（流式输出）
- **GET** `/predictVideo?username=...&weight=...&conf=...&startTime=...&inputVideo=...`
- **返回**：`multipart/x-mixed-replace`（浏览器以 `<img :src="...">` 的方式显示“连续帧”）
- **结束后**：
  - Flask 转码并上传结果视频到 Spring `/files/upload`
  - Flask 调 `POST http://localhost:9999/videoRecords` 写入记录
  - Flask 通过 Socket.IO 推送 `message/progress`

### 5.5 摄像头预测（流式输出）
- **GET** `/predictCamera?username=...&weight=...&conf=...&startTime=...`
- **返回**：同上（连续帧）
- **停止**
  - **GET** `/stopCamera`：设置录制标记为 false，触发保存流程

---

## 6. WebSocket（Socket.IO）

- **连接地址**：`http://localhost:5000`  
  前端封装在 `CropDiseasesDetection_vue/src/utils/socket.ts`

- **事件**：
  - `message`：Flask `emit('message', {data: '...'})`，前端会提示弹窗
  - `progress`：视频/摄像头保存时的进度百分比（0-100）

---

## 7. “调用链”总结（按功能）

### 7.1 单图检测（会入库 imgRecords）
1. 前端上传图片 → `POST http://localhost:9999/files/upload`（Element Upload）
2. 前端请求预测 → `POST /api/flask/predict` → 实际 `POST http://localhost:9999/flask/predict`
3. Spring 转发 → `POST http://localhost:5000/predictImg`
4. Flask 推理 & 上传结果图 → `POST http://localhost:9999/files/upload`
5. Spring 将推理结果写入 MySQL → `/imgRecords` 表
6. 前端解析 `res.data`（字符串 JSON）并渲染

### 7.2 批量检测（当前代码不入库记录）
1. 前端选择文件夹 → 打 zip → `POST http://localhost:9999/files/uploadFolder`
2. 前端调用批量预测 → `POST /flask/predictImgBatch` → 实际 `POST http://localhost:5000/predictImgBatch`
3. Flask 下载 zip → 批量推理 → 直接把结果数组返回前端

### 7.3 视频检测（入库 videoRecords）
1. 前端上传视频 → `POST http://localhost:9999/files/upload`
2. 前端拼接流地址 → `http://127.0.0.1:5000/predictVideo?...`
3. Flask 处理 → 上传结果视频 → `POST http://localhost:9999/files/upload`
4. Flask 写入记录 → `POST http://localhost:9999/videoRecords`
5. 前端用 Socket.IO 接收保存进度

### 7.4 摄像头检测（入库 cameraRecords）
同视频，流地址为 `http://127.0.0.1:5000/predictCamera?...`，停止用 `/flask/stopCamera`（Vite 代理到 Flask）。
